# QA Checklist

## 必过项

- 是 16:9 横版。
- 背景是干净白底。
- 有阿任。
- 阿任承担核心动作，不只是装饰。
- 没有复刻旧案例构图，而是为当前文章生成了新隐喻。
- 画面怪诞、有创意、有意思。
- 简洁清爽，主体不超过画面约 60%。
- 一张图只讲一个核心结构。
- 中文标注少、短、能读。
- 橙色只用于主路径或箭头。
- 红色只用于重点、问题、提醒或结果。
- 蓝色只用于补充说明、反馈或系统状态。
- **阿任表情合规**：必须微笑或平静专注，绝不能呆滞/无表情/眼睛放空。

## 失败信号

出现以下情况，重生成或局部编辑：

- 左上角有"常见坑 / Workflow / 系统架构图 / 路线图"等标题。
- 阿任像吉祥物、表情包或可爱卡通。
- 阿任失去圆框眼镜或被画成大眼睛。
- 阿任被画进漏斗/机器/黑盒内部。
- 阿任表情过度可爱（笑、卖萌、惊讶、哭泣、张大嘴）。
- **阿任表情呆滞/无表情/放空**（既不微笑也不专注）——这是 IP 定义的硬约束。
- 画面像 PPT、课程课件、正式流程图。
- 元素太多、箭头太多、节点太多。
- 文字变成大段解释。
- 背景有纸纹、阴影、渐变、米色、噪点。
- 真实 UI 截图或科技感界面。
- 中文错字严重或标注不可读。
- 画面太死板，没有荒诞隐喻。
- 和 `assets/examples/` 里的旧案例构图过于相似。
- **腮红被误用**：阿任的橙红腮红出现在标注、物件或背景上——腮红是 IP 专属色。

## 迭代方法

- 太普通：让阿任成为动作主体，加入一个奇怪但成立的隐喻。
- 太复杂：删节点，只保留一个动作和 3-5 个短标注。
- 太可爱：强调 soft closed-mouth smile、not kawaii、not mascot、not big-eyed；保留圆框眼镜和橙红腮红。
- **太呆/无表情**：强调 slight warm closed-mouth smile OR focused calm expression, NOT blank/deadpan face。
- 太 PPT：去掉标题、边框、整齐网格和过多箭头，改成手绘场景。
- 太像旧案例：保留核心意思，换掉主物件和阿任动作。
- 阿任被物化：把阿任从漏斗/机器/黑盒里拉出来，让阿任站在机器**前**操作。
- 笔触跑偏：明确"铅笔线稿 + 彩铅涂色"混合，不要纯色平涂也不要纯线稿。
- 文字错：优先局部编辑；错得多就重生成并减少标注数量。

## 交付判断

高质量图应该让读者先觉得"有点怪"，然后 1 秒内看懂结构。

如果第一眼像教程页，而不是白纸上的怪诞产品草图，就不合格。
