# 构图模式与原创规则

## 基础结构类型

选择一种结构即可，不要混太多。

### Workflow 流程

适合：输入 -> 处理 -> 输出，内容生产，AI 工作流，自动化链路。

画法：左侧输入，中间阿任或怪机器处理，右侧输出，橙色箭头表达主流向。

### 系统局部

适合：信息来源、过滤器、数据库、渲染器、agent 系统局部。

画法：只画 3-5 个核心模块，阿任参与其中一个关键动作。

### 前后对比

适合：混乱/有序，手动/自动，分散/收拢，焦虑/稳定。

画法：左混乱，右稳定，中间橙色箭头。角色可以更夸张。

### 角色状态

适合：用户痛点、创作者状态、工具太多、信息焦虑、卡住到跑起来。

画法：2-4 个小状态，每个状态一个短标注。

### 概念隐喻

适合：内容工厂、信息仓库、脑内黑盒、工作流机器、自动日报。

画法：一个大的怪物件或机器，少量输入，一个输出。要有记忆点。

### 方法分层

适合：方法论框架、系统层级、能力栈、内容系统分层。

画法：一层层盒子，不要正式金字塔；阿任在旁边搬砖或搭建。

### 地图路线

适合：从想法到上线、用户路径、内容承接路径、学习路线。

画法：一条弯曲路径，少量节点，阿任牵线或走路。

### 小漫画分镜

适合：失败到成功、真实过程、吐槽、使用前后变化。

画法：2-4 个小场景，每格只表达一个动作。

## 原创隐喻生成法

每次都从当前文章重新发明隐喻，不能照搬旧图。

### 三步

1. 把抽象概念换成一个物理动作：卡住、漏掉、变重、分拣、沉淀、发酵、开门、折叠、拆包、回流。
2. 把系统结构换成一个低科技物件：坏掉的机器、纸箱、抽屉、水管、邮筒、怪表盘、秤、井、梯子、奇怪工位。
3. 让阿任承担动作：不是站旁边看，而是**站在机器前操作**、拉错线、守在门口、搬运、修补、称重、扶梯子、记录、把东西塞进某个怪装置、调试按钮、翻看笔记。
   - **重要约束**：阿任是真人比例的少年，**可以**被画进漏斗/机器/黑盒内部（承担"被裹挟"、"钻进查看"、"卡在中间"等戏剧化动作），但**始终保持 Q 版 1:2.5 真人比例和默认外貌**。阿任可以进设备，**不能**被物化成零件/齿轮/沙漏一粒/线团一团。

### 可用物件池

- 纸箱、抽屉、旧机器、漏斗、秤、邮筒、门、井、梯子、水管、线团、闸门、转盘、黑盒、打孔器、压面机、晾衣绳、怪工位。
- 阿任气质专属：笔记本、咖啡杯、相机、旅行箱、椅子、书桌、平板电脑、画板。
- 用时只选 1-2 个，不要堆满。

### 阿任动作池

- **操作类**：按、拧、推、调试、查、改、点、输入、勾选、切换。
- **搬运类**：扛、搬、推车、接、塞、压、称、分拣。
- **路径类**：走、牵、跑、停、看、卡住、堵在、扶、绕。
- **记录类**：翻、记、写、贴、盖章、批注、画、圈。
- 动作要服务核心意思，不要为了怪而怪。
- 避免把阿任画成"被物化"的状态（如：被吸进漏斗、卡在黑盒内部、变成沙漏的一粒沙）。

## 反复刻规则

不要默认打开或复刻 `assets/examples/`。这些图片只用于风格校准，例如线条密度、留白、颜色克制、阿任气质。

除非用户明确说"照这张 / 复刻这个构图 / 用这个案例改"，否则不要直接复用这些旧构图（**这些是原小黑 skill 的旧构图，对阿任 IP 全部需要重新设计**）：

- 传送带两个断点
- 阿任在内容机器**前**拉判断杆（注意不是"在机器里"）
- 阿任**用**漏斗分拣流量/信任/转化（注意不是"变漏斗"）
- 阿任切素材鱼
- 阿任牵承接路径
- 阿任拉三层信息源
- 三个阿任分别拿喇叭/搭桥/开门
- 阿任盖章话术工具箱
- 阿任举牌看常见坑路径

同类主题也要换新隐喻。例如"承接路径"不一定画路线，可以画阿任把内容尾巴接到门把手；"一鱼多吃"不一定画鱼，可以画阿任把一个纸团压成几种形状；"信息过滤"不一定画漏斗，可以画阿任在分拣一堆明信片。

**阿任专属约束（2026-07-13 更新）**：
- ✅ 阿任**可以**被画进漏斗/机器/黑盒**内部**——承担"被裹挟"、"钻进查看"、"卡在中间"等戏剧化动作。
  - 前提：进设备时**保持 Q 版 1:2.5 真人比例 + 默认外貌**（圆框眼镜 + 蓬松发 + OS 夹克 + 橙红腮红）。不能缩成 mini，不能被物化成零件。
- ❌ 让阿任"变成"某种物件（变成沙漏、变成线团、被物化）。
- ❌ 让阿任坐在机器零件上像骑玩具。
- ❌ 给阿任加翅膀、魔法、机甲等超现实元素。
- ❌ 把阿任的圆框眼镜去掉——这是 IP 最强识别符。
