# 风格 DNA

## 一句话

纯白、极简、彩铅+铅笔混合手绘、留白、克制、怪诞、产品草图感、中文手写感、结构清楚但不说明书。

像一个长期做产品、设计、AI 工具的青少年，在白纸上随手画出来的一张解释草图。

## 必须

- 16:9 横版正文配图。
- 纯白背景：不要米色、暖灰、纸张纹理、渐变、阴影、噪点、复古纸感。
- **彩铅+铅笔混合手绘**：
  - 线稿用铅笔（细线、轻微抖动、不机械、不矢量、不厚重描边）
  - 涂色用彩铅（皮肤、衣物、物品有微妙彩铅涂色感，不是平涂）
- 大量留白：主体占画面约 40%-60%，至少 35% 空白，最好有一整块安静区域。
- 少量中文手写批注：最多 5-8 处，每处尽量 2-8 个字。
- 一张图只讲一个核心动作、结构、状态或隐喻。
- 结构要自然表达，不要在图上写结构类型名称。
- IP 阿任（新海诚动画风格 + 彩铅手绘少年）必须参与核心动作，详见 `ip-character.md`。

## 颜色（5 色制，**严格区分**）

| 颜色 | 用途 | 谁用 |
|---|---|---|
| **黑色** | 主体线稿、阿任的头发/眼镜框/夹克/裤/鞋、文字、框线、结构 | 阿任和标注 |
| **橙红色** | **仅限**阿任的腮红（IP 识别符，固定不变） | 阿任专属 |
| **红色** | 警示、问题、关键提醒、批注、情绪点、结果 | 中文标注专用 |
| **橙色** | 主流程、路径、箭头、自动化流向、从 A 到 B 的移动关系 | 中文标注专用 |
| **蓝色** | 补充说明、脑内状态、系统状态、第二层解释、AI/assistant 提示 | 中文标注专用 |
| **微妙暖色** | 阿任的皮肤彩铅涂色 + 物品（咖啡、纸杯、桌面） | 仅限阿任本身和物品 |

### 关键规则

- **腮红和警示标注必须用不同色**：腮红 = 橙红（专属），警示 = 红色（标注）。绝不能混用。
- **背景禁止任何颜色**：纯白就是纯白，米色/暖灰/纸纹都不行。
- **每张图标注颜色不超过 3 种**（红/橙/蓝里选），保持克制。
- 蓝色不是每张都必须用，宁可少不要多。

## 绝对不要

- 不要商业插画、PPT 信息图、正式流程图、课程课件。
- 不要可爱卡通海报、儿童插画、复杂架构图、精致扁平插画。
- 不要科技感 UI、真实 App 截图、复杂背景、渐变、阴影、纹理。
- 不要把每个节点都解释清楚，不要大段文字型信息图。
- 不要左上角写"Workflow 流程图 / 系统架构图 / 常见坑 / 路线图"等类型标题。
- 不要让阿任的脸/表情包抢过结构。
- 不要在背景出现米色/暖灰/纸纹（阿任的腮红和肤色不算背景）。
- 不要让阿任呆滞或面无表情——要么微笑，要么平静专注。

## 审美方向

要怪诞、创意、有意思、简洁清爽、天马行空。

不要可爱、幼稚、很复杂、死板、商业精致。

## 阿任 vs 小黑的视觉差异

| 维度 | 小黑（旧） | 阿任（新） |
|---|---|---|
| 形象 | 纯黑色实心小怪物 | 真实感强的彩铅手绘少年 |
| 黑色占比 | 占全身 100% | 占头发+夹克+裤+鞋的 60% 左右 |
| 主色调 | 黑+白 | 黑+白+橙红腮红 |
| 笔触感 | 单色线稿 | 铅笔线稿 + 彩铅涂色 |
| 表情 | 完全空白、呆 | 温和微笑或平静专注（绝不能呆滞） |
| 气质 | 荒诞冷幽默 | 认真书卷气，带一点怪诞 |
| 动作风格 | 系统操作员 | 文艺青年 + 工程师混合 |

**重要：** 换 IP 不只是换脸。原来的"纯黑小怪物"可以塞进任何机器、漏斗、黑盒里当零件；阿任是真人比例的少年，更接近"团队里那个戴眼镜的同事"——所以构图时不要把他物化（不要塞进机器里当零件），要让他"在操作机器"。
