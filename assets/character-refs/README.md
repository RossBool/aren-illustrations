## 表情硬约束

- ✅ 微笑 / 平静专注 / 轻严肃有神
- ❌ 皱眉 / 严肃无表情（08 / 10 是反面参考）
- ❌ 大笑 / 卖萌 / 惊讶 / 哭泣 / 呆滞（已写入 `references/qa-checklist.md`）

## AI 生图时的使用

参考这些图确保"阿任永远像阿任"：

1. **形象一致性**：脸型 / 眼镜 / 腮红 / 发型 / 夹克——5 个核心元素不能变形
2. **动作扩展**：走路、坐下、持物、侧面——避免动态变形
3. **服装变体**：hoodie / 简单夹克——IP 服装可以变，5 个核心元素不能丢
4. **反面参考**：08 / 10 告诉 AI 这是 IP 表情下界——**不要学**

## Multi-image input（`input_files`）用图指引

> **核心问题**：MCP image generation 接收 prompt 时是"一次性、瞬时性"的——单纯文字描述会关联到通用动漫 IP，IP 跑偏 50-70%。
>
> **解决办法**：用 `input_files` 传 2-3 张 character-refs 作**视觉锚**——让 MCP 模型"看到"阿任长啥样。
>
> **实测效果**（2026-07-12 验证）：IP 还原度从 ~30% 提升到 ~65%。

**推荐 3 张**：
- `01-base-front.png`（基线正面，OS 短款黑夹克标准站姿）
- `04-walking.png`（动态，验证下身走姿）
- `12-jacket-simple.png`（OS 简单夹克 + 微笑基线）

**不要传**：
- ❌ **校准图**（13-aren-curved-path.png 等）——带"豆包AI生成"水印 + 风格与目标不同
- ❌ **11-trench-coat.png**（长款风衣不是阿任默认款，会污染生成）
- ❌ **08/10 严肃脸**（反面参考，会让 AI 学到严肃表情）

> 全部图片由豆包 AI 生成，保留原图水印"豆包AI生成"作为来源标识。
