# 校准样图与形象参考

本目录分两类资产：

- `assets/examples/`：**校准图**——AI 生图时作为"目标效果"视觉锚点；按 8 种结构类型组织。
- `assets/character-refs/`：**形象参考库**——阿任 IP 在不同角度、不同动作、不同场景下的标准立绘；AI 生图时作为"形象一致性"参考。

> 全部图片由 **豆包 AI** 生成（`01-aren-carrying-box.png` / `02-aren-debugging-funnel.png` 例外，由本 skill 通过 MCP 图像生成 API 制作）。保留原图水印"豆包AI生成"作为来源标识。

### 校准图来源说明

- **01 / 02** 由 MCP（`matrix_generate_image`）生成——作为 IP 形象锚点，**形象完美**（与 v1 原图一致），但 02 表情略呆（详见下方"已知问题"）。
- **03 ~ 12** 由豆包 AI 直接生成（用户预先制作）——作为"目标效果"参考，**风格统一、覆盖全**，但 skill 提示词**不保证能复现同等质量**。

AI 生图时会参考这些图的线条密度、留白比例、IP 气质，作为视觉锚点。但**实际生成质量取决于 MCP 的图像生成能力**——这是 skill 的天花板，不是校准图不够好。

### 校准图 02 状态说明（保留原因）

**02 的表情略"呆"**——阿任 IP 定义要求"温和微笑或平静专注"，但 02 的实际表情介于"呆滞"和"专注"之间。原因：用 text-to-image 重画表情时，AI 把发型/服装也改动了，无法做到"只修表情保留其余"。MCP 当前没有局部编辑（inpaint）能力。

**为何保留 02 不删**：02 的核心价值是"IP 形象锚点"——阿任的发型、脸型、眼镜、夹克、整体气质在 02 里**和 v1 原图完全一致**，是 AI 生图时"参照 IP 长什么样"的关键参照。表情问题已在 `references/qa-checklist.md` 列为失败信号，实际生图时会强制约束。

**未来可能的优化**：
- 用 PS / Figma 手改 02 表情（保留 v1 形象 + 修好表情）
- 等 MCP 出现 inpaint 能力后再用 image-to-image + 局部 mask
- 用 12 张豆包AI校准图替换 01 02（如要换，优先选 02 表情最稳的一张作"形象锚点"）

### 形象参考库的用途

AI 生图时会参考这些图确保"阿任永远像阿任"：

1. **形象一致性**：脸型、眼镜、腮红、发型、夹克——这 5 个元素在不同角度/动作下不能变形
2. **动作扩展**：走路、坐下、持物、侧面——AI 生成新动作时参照这些，避免动作变形
3. **服装变体**：hoodie、风衣、简单夹克——IP 服装可以变，但 5 个核心元素不能丢
4. **反面参考**：08 / 10 展示了"严肃 / 皱眉"表情——**告诉 AI 这是 IP 的下界，不要学**

### 表情硬约束

- ✅ **允许**：微笑（基础 / 09 / 12 / 11 / 06 / 04 / 01）、平静专注（07 / 05 / 02）、轻严肃但有神（罕见，参考 01 02 校准图）
- ❌ **避免**：皱眉 / 严肃无表情（08 / 10）——IP 定义要求"温和微笑或平静专注"
- ❌ **禁止**：大笑、卖萌、惊讶、哭泣、呆滞——已写入 `references/qa-checklist.md`

## 命名规范

### 校准图

`0N-aren-{动作或类型}.png`，N 是两位数字序号，{动作或类型} 用英文短横线连接。

例：`01-aren-carrying-box.png` / `06-aren-method-layering-stack.png` / `12-aren-storyboard-4panel.png`

### 形象参考

`NN-{角度或状态}.png`，NN 是两位数字序号。

例：`01-base-front.png` / `03-back.png` / `11-trench-coat.png`

## 何时需要补图

- 校准图：AI 生成的某种结构类型图经常跑偏（线条太重/留白太少/IP 不像）
- 形象参考：阿任在某个新角度/新动作下严重变形，需要补一张锚定
- 用户反馈"这个角度/结构类型的图都不像"
- 每次新加结构类型时补一张校准图

## 临时方案

- 全部 8 种结构类型已有校准图，无需临时方案
- 实际生成质量由 QA 阶段用 `references/qa-checklist.md` 校验
